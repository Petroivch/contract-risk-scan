# UX-документация — Contract Risk Scanner (мобильная версия)

## Область пакета
Эта папка содержит UX-артефакты для мобильного продукта `Contract Risk Scanner` на `Android` и `iPhone`.

В пакет входят:
- карта экранов и пользовательские потоки;
- матрица состояний экранов;
- спецификация компонентов;
- правила microcopy и локализации;
- handoff-требования для frontend/backend;
- стандарты качества UX/UI;
- визуальное направление;
- hi-fi спецификация по ключевым экранам.

## Контекст продукта
Пользователь загружает файл договора и получает:
1. список рисков;
2. спорные или неоднозначные пункты;
3. краткое summary договора с акцентом на выбранную роль.

Перед загрузкой пользователь выбирает роль через редактируемый dropdown:
- предустановленные варианты: `Contractor` / `Employer`;
- допускается собственная роль.

## Ограничения продукта
- релиз только для мобильных платформ: `Android` и `iOS`;
- после установки пользователь не должен скачивать дополнительные модули или модели;
- UX должен быть `local-first`: история и открытые отчеты доступны локально;
- все пользовательские тексты должны поддерживать `ru`, `en`, `it`, `fr`, при этом `ru` — язык по умолчанию и fallback.

## Контракт между документами
- `01` определяет карту экранов, основные потоки и правила навигации.
- `02` определяет состояния экранов и fallback-поведение.
- `03` определяет контракт компонентов и токенов.
- `04` определяет i18n-правила, ключи и тон текста.
- `05` определяет handoff для frontend/backend и интеграционные гейты.
- `06` определяет константы и стандарты качества.
- `07` определяет визуальную рамку и анти-шаблонные ограничения.
- `08` определяет hi-fi поведение экранов и правила имплементации.

Порядок приоритета при конфликте документов:
`06` -> `04` -> `03` -> `01/02` -> `05` -> `07` -> `08`

## Принципы дизайна
- Юридический смысл должен считываться за `3–5` секунд.
- Выбранная роль должна быть заметна во всех критичных точках сценария.
- У каждого асинхронного шага есть понятный прогресс, fallback и путь восстановления.
- Визуально продукт должен выглядеть как премиальный legal-tech инструмент, а не как generic SaaS-форма.
- Базовая доступность обязательна: touch target `>= 44px`, контраст `WCAG AA`, поддержка динамического текста.
- Никакого хардкода: только `i18n keys`, `design tokens`, `constants registry`.
