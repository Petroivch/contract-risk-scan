# План оптимизации размера frontend-сборки

## Бюджет
- Общий лимит всего проекта: `228 МБ`
- Целевой вклад mobile/frontend: `<= 120 МБ`

## Что влияет на размер mobile
- Expo / React Native runtime
- native зависимости
- иконки, изображения, шрифты
- локализации
- встроенный JS bundle
- debug/release различия Android build

## Что уже важно учитывать
- в приложении только нужные локали: `ru/en/it/fr`
- не тащить тяжелые ассеты без необходимости
- не хранить debug-only зависимости в release path
- избегать лишних UI-библиотек, если тот же результат можно собрать своими компонентами

## Практические меры
1. Assets
   - убрать неиспользуемые изображения;
   - сжимать растровые ассеты;
   - не раздувать набор шрифтов.
2. JS bundle
   - вычищать мертвые зависимости;
   - не добавлять крупные utility libs без необходимости.
3. Android
   - держать Hermes включенным для release;
   - использовать minify / shrinkResources в release pipeline;
   - при необходимости выпускать split ABI артефакты.
4. i18n
   - хранить только четыре обязательные локали;
   - не дублировать большие текстовые демо-ресурсы.

## Если итог > 228 МБ
1. split ABI артефакты;
2. агрессивнее сжать ассеты и шрифты;
3. заменить тяжелые зависимости;
4. вынести необязательный demo-контент из bundle;
5. еще жестче разделить debug и release pipeline.

## Контроль
- проверять размер Android/iOS артефактов на каждом релизном прогоне;
- блокировать релиз, если сумма артефактов и обязательных deliverables превышает `228 МБ`.
